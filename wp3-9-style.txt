<h2>H2 Header: The Most Common Header I Use</h2>
I'm trying to write as little HTML as possible when I write posts in WordPress. My control freak nature means I will probably still edit in HTML mode fairly often, but I'm trying to keep it to a minimum so I can stay in touch with how non-coders use WordPress and other CMSes.
<ul>
	<li>Unordered lists are pretty common, especially on websites that are trying to sell things.</li>
	<li>I use them a lot because they're easy to 'digest' quickly.
<ul>
	<li>Nesting these is kind of terrible, but I still allow for it in my theming</li>
	<li>Just in case I decide to write a post that's 'kind of terrible'.</li>
</ul>
</li>
	<li>They also break up long blocks of text nicely when you don't have a graphic or other media element to do that for you.</li>
</ul>
<h3>H3 Header: Because Some Posts Go Really In-Depth and Need Hierarchical Structure in their Headers</h3>
I try not to make my posts so complicated that I need to use H3s, but it happens.
<ol>
	<li>I only use Ordered Lists if I'm explaining steps in a process or giving a particular number of reasons for something.</li>
	<li>If I'm using it to list reasons, I'm probably really excited about how many justifications I can make for my viewpoint.</li>
	<li>If I'm using it for a process, and it goes into sub-lists:
<ol>
	<li>I should probably just make a Flow Chart Image instead</li>
	<li>Because I can make this look kind of pretty</li>
	<li>But it is obviously a complex concept that could be better represented in other ways.</li>
</ol>
</li>
	<li>???</li>
	<li>Profit</li>
</ol>

<h2>Quotes</h2>
I used to code my own Quote styling into each post, which was really annoying. Luckily WordPress has improved vastly since then and I should adjust my posts to suit this progress.
<blockquote>This is how quotes look. I wanted to make it so I could cite a source for my quotes without going into the code tab and adding in &lt;cite&gt; manually. <cite>Sadly, I'll still have to go in and code that <a href="#">for now</a></cite></blockquote>
Using a soft return (Shift+Enter) and the <del>strikethrough</del> style button will have to do for now. I will have my theme look for that tag and replace it with &lt;cite&gt; on page load using AJAX. I chose &lt;del&gt; because it's <strong>highly</strong> unlikely I'll ever want to use it for a citation. It's not elegant, but it solves my usability issue.
